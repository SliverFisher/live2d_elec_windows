### 问题 3：flatten-release.ps1 中 Copy-Item -Recurse -Force 静默跳过被锁定文件

**现象**：第二次 `npm run package` 完成（exit code 0）后，检查发布目录：

| 文件 | 大小 | LastWriteTime |
|---|---|---|
| `resources\app.asar` | 29,567,356 字节 | 2026/7/5 19:36:06 |
| `Live2DRenderer.exe` | 10,666,564 字节 | 2026/7/5 20:10:45 |
| `Live2DRendererHost.exe` | 204,521,984 字节 | 2026/7/5 20:10:40 |
| 其他 dll/pak 文件 | - | 2026/7/5 20:10:xx |

`app.asar` 的时间戳（19:36:06）明显早于其他文件（20:10:xx），且大小为 29,567,356（旧值），而新构建的 `dist/win-unpacked/resources/app.asar` 大小为 29,568,613。

**直接原因**：[flatten-release.ps1 L31-L34](file:///C:/Users/49213/Desktop/A/codex/Live/src/Live2DRendererElectron/scripts/flatten-release.ps1#L31-L34)（原值）使用：

```powershell
Get-ChildItem -LiteralPath $unpackedRoot -Force | ForEach-Object {
  $targetPath = Join-Path $releaseRoot $_.Name
  Copy-Item -LiteralPath $_.FullName -Destination $targetPath -Recurse -Force
}
```

`Copy-Item -Recurse -Force` 在目标文件被锁定时不会抛出异常（`$ErrorActionPreference = 'Stop'` 未触发），而是静默跳过该文件。整个脚本以 exit code 0 完成，但 `app.asar` 实际未被覆盖。

**手动验证**：

```powershell
Copy-Item -LiteralPath $distAsar -Destination $releaseAsar -Force -ErrorAction Stop
```

这次成功，`app.asar` 更新为 29,568,613 字节，时间戳变为 20:10:39。说明在 `Get-ChildItem` 管道上下文中 `Copy-Item -Recurse -Force` 的行为与单独调用不同。

### 问题 4：PowerShell 脚本语法错误导致清理中断

**命令**：

```powershell
@('builder-debug.yml', 'builder-effective-config.yaml') | ForEach-Object {
  $f = Join-Path $releaseDir $_
  if (Test-Path $f) {
    try {
      Remove-Item $f -Force -ErrorAction Stop
      Write-Host "Removed $_"
    } catch {
      Write-Host "Could not remove $_: $_"   # ← 错误行
    }
  }
}
```

**错误**：

```
At line:49 char:36
+       Write-Host "Could not remove $_: $_"
                                    ~~~
Variable reference is not valid. ':' was not followed by a valid variable name character.
Consider using ${} to delimit the name.
+ CategoryInfo          : ParserError: (:) [], ParentContainsErrorRecordException
+ FullyQualifiedErrorId : InvalidVariableReferenceWithDrive
```

**原因**：双引号字符串内 `$_:` 被 PowerShell 解析为驱动器引用（drive reference），而非变量 `$(_)` 后接冒号。需写成 `${_}:` 或使用单引号。

**影响**：清理脚本中断，exit code 1。`win-unpacked.tmp`、`builder-debug.yml` 等残留文件未被删除。

## 四、修复措施

### 4.1 electron-builder.yml 输出目录改为 dist

**文件**：[electron-builder.yml L4-L5](file:///C:/Users/49213/Desktop/A/codex/Live/src/Live2DRendererElectron/electron-builder.yml#L4-L5)

```yaml
# 修改前
directories:
  output: ../../release/MaidAI/Live2DRenderer

# 修改后
directories:
  output: dist
```

electron-builder 不再直接写入发布目录，避免与已锁定的旧文件冲突。

### 4.2 flatten-release.ps1 改用 robocopy

**文件**：[flatten-release.ps1 L28-L56](file:///C:/Users/49213/Desktop/A/codex/Live/src/Live2DRendererElectron/scripts/flatten-release.ps1#L28-L56)

将 `Copy-Item -Recurse -Force` 替换为 `robocopy /E /IS /IT /R:5 /W:2`：

- `/E` 复制子目录（包括空目录）
- `/IS` 覆盖相同文件
- `/IT` 覆盖仅属性变化的文件
- `/R:5` 最多重试 5 次
- `/W:2` 每次重试间隔 2 秒
- exit code < 8 视为成功

### 4.3 手动覆盖 app.asar

在 flatten-release.ps1 静默跳过后，手动执行 `Copy-Item -LiteralPath ... -Force` 成功覆盖了 `app.asar`。
